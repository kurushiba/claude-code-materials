Hooksの6つの保存場所（スコープ）

  • ユーザーレベル: ~/.claude/settings.json
    自分専用のルール（通知など）
  • プロジェクトレベル: .claude/settings.json
    チームで共有するルール（Linter実行など）
  • ローカルレベル: .claude/settings.local.json
    Git共有しない、自分だけのプロジェクト固有ルール

※他にプラグイン、Skillフロントマター（呼び出し時のみ有効）、マネージドポリシー（全社強制）があります。

Hooksの基本構文と設定

  • hooksキーの下にイベント名（PreToolUse等）ごとに配列で定義
  • matcher: 絞り込み条件（ツール名など）
  • type: command（シェルコマンド実行）が基本
  • データ連携: フック発火時、スクリプトの標準入力にツール情報がJSON形式で渡される
{
  "hooks": {
    // どのタイミングで？ (イベント名)
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "if": "Bash(rm *)",
            // 何を実行する？ (コマンド)
            "command": "${CLAUDE_PROJECT_DIR}/.claude/hooks/block-rm.sh"
          }
        ]
      }
    ]
  }
}

Matcherによる実行条件の絞り込み

  • 無関係な処理でフックが走るのを防ぐ「ふるい」の役割
  • 完全一致: 英数字とパイプ記号
    例: "Edit|Write"（EditかWriteツール）
  • 正規表現: 記号（ドットや*等）を含む場合
    例: "mcp__.*"（mcp__で始まるツール）

より詳細な条件指定「if構文」

  • ツールの引数（中身）まで見て発火を制御ツールイベント限定
  • 書き方: "Bash(git *)" や "Edit(*.ts)"
  • 複合コマンド（npm test && git push）も自動分割してマッチ可能
  • スクリプトの無駄な起動を防ぐため、処理速度が向上する
Matcherのみ
"Bash"
✗ ls (起動して無視)
✗ echo (起動して無視)
✓ git push (起動して実行)
毎回スクリプトが起動し遅い
if構文
"Bash(git *)"
ー ls (起動しない)
ー echo (起動しない)
✓ git push (起動して実行)
必要な時だけ起動し速い

Claudeを待たせない「非同期実行」

  • 設定に "async": true を追加
  • 裏で最大 timeout 秒間スクリプトを走らせ、Claudeの作業をブロックしない
  • 注意点: ツールの実行を途中で止められないため、危険なコマンドのブロックなど安全用途には使えない
  • asyncRewake: 処理失敗時にClaudeを自動再起動
