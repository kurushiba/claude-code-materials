Claude Codeの「Hooks（フック）」とは？

  • 特定のタイミングで
    自動的に処理を走らせる仕組み
  • AIの判断に頼らず、
    システムレベルで強制ルールを適用
  • 「お願い」ではなく「絶対のルール」

設定のイメージ（settings.json）

  • AIへのプロンプトではなく、
    設定ファイルに直接記述します
  • 「どのイベントで」
    「どのコマンドを実行するか」を定義
  • 細かい条件（Matcher）も設定可能
{ "hooks": { // どのタイミングで？ (イベント名) "PreToolUse": [ { "matcher": "Bash", "hooks": [ { "type": "command", "if": "Bash(rm *)", // 何を実行する？ (コマンド) "command": "${CLAUDE_PROJECT_DIR}/.claude/hooks/block-rm.sh" } ] } ] } }

なぜHooksが必要？（CLAUDE.mdとの違い）

CLAUDE.md / Skill

「.envファイルは編集しないでね」

AI「了解！でも会話が長くなって忘れたから、必要そうだし編集しちゃおう…」

❌ ルールが破られる可能性あり
🛡️
Hooks
（絶対のルール）

システム「編集イベントは絶対ブロック」

AI「編集するぞ」
→ Hooks「ダメです。処理を止めます。」

✅ 確実に処理が止まる・実行される

Hooksを使う3つのメリット

🛡️
1. 安全性を機械的に守る
ファイルを丸ごと消す危険なコマンドや、パスワードファイルの編集を強制的にブロック。安心してAIに任せられます。
✨
2. コード品質を自動維持
ファイル編集直後にPrettierなどの整形ツールを自動実行。誰が書いても同じ綺麗なスタイルに揃えられます。
🤖
3. 自律モードで進めやすい
「安全の柵」と「品質ゲート」があるため、人間が毎回確認しなくても、AIにまとまった作業を自律的に任せやすくなります。

押さえておきたい4つのイベント

セッション中の発生タイミング
SessionStart
セッション開始・再開時
（初期化・情報注入）
ブロック可能
PreToolUse
ツール実行の直前
（危険操作のせき止め）
PostToolUse
ツール実行の直後
（コード整形・履歴記録）
関門
Stop
AIが作業を終える直前
（テスト自動実行など）
※その他にも PermissionRequest, UserPromptSubmit など計10種類のイベントが存在します。
5 / 6

代表的な4つのユースケース

🚫
PreToolUse
危険な操作のブロック
rm -rf などのファイル削除や、.git フォルダへのアクセスを実行前にせき止める。
🧹
PostToolUse
コード自動整形・履歴記録
ファイル編集直後にPrettierを走らせたり、ツール実行履歴をファイルに書き出す。
✅
Stop
テストが通るまで終わらせない
終了前に npm test を実行。失敗すればエラーを返し、AIに強制的に修正させる。
📡
SessionStart 等
最新情報を自動で渡す
現在のGitブランチ名や最新のCIエラーなどを取得し、AIのコンテキストに注入する。
